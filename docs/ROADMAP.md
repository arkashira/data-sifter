# ROADMAP.md

## Roadmap

The Data Sifter project aims to deliver a visual workflow builder for data scraping. This roadmap outlines the key milestones and features for the project.

### MVP Milestone (Must-Have for Launch)

The MVP milestone focuses on delivering a functional and usable product that meets the core requirements.

#### MVP Critical Features

*   **Workflow Creation**: Users can create new workflows by adding blocks to the canvas.
*   **Block Connections**: Users can connect blocks to create a workflow.
*   **Workflow Saving**: Users can save workflows to a JSON file.
*   **Workflow Loading**: Users can load workflows from a JSON file to execute them.

#### MVP Deliverables

*   A functional workflow builder with a canvas for adding and connecting blocks.
*   A JSON file format for saving and loading workflows.
*   Basic error handling and logging for workflow execution.

### v1 Phase (Should-Have Features)

The v1 phase focuses on enhancing the user experience and adding features that improve the overall usability of the product.

#### v1 Features

*   **Block Library**: A library of pre-built blocks for common data scraping tasks.
*   **Block Customization**: Users can customize block properties and behavior.
*   **Workflow Validation**: The system validates workflows for errors and inconsistencies before execution.
*   **Execution History**: Users can view the execution history of workflows.

#### v1 Deliverables

*   A block library with a minimum of 10 pre-built blocks.
*   Block customization options for properties and behavior.
*   Workflow validation and error handling.
*   Execution history feature.

### v2 Phase (Nice-to-Have Features)

The v2 phase focuses on adding advanced features and improving the overall performance of the product.

#### v2 Features

*   **Machine Learning Integration**: Integration with machine learning models for data enrichment and analysis.
*   **Real-time Data Processing**: Real-time data processing and visualization.
*   **Collaboration Features**: Users can collaborate on workflows and share them with others.
*   **Advanced Block Types**: Support for advanced block types, such as web scraping and API integration.

#### v2 Deliverables

*   Integration with machine learning models for data enrichment and analysis.
*   Real-time data processing and visualization.
*   Collaboration features for workflow sharing and collaboration.
*   Support for advanced block types.

## Next Steps

The next steps for the Data Sifter project are to focus on delivering the MVP milestone. This will involve prioritizing the MVP critical features and deliverables, and making sure that the product meets the core requirements. Once the MVP is delivered, the project will move on to the v1 phase, focusing on enhancing the user experience and adding features that improve the overall usability of the product.
